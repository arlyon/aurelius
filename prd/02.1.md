# Product Requirements Document (PRD): Aurelius 2.0 - "Metabolic Synthesis"

**Version:** 2.2-COMPREHENSIVE  
**Status:** Approved for Development (April 2026 Target)  
**Primary Language:** Rust  

## 1. Introduction / Overview
Aurelius 2.0 transitions the system from a static Retrieval-Augmented Generation (RAG) CLI into a dynamic, self-curating **World Model** and High-EQ partner. 

Powered by **Qwen 3.6 (35B, 4-bit Quantized)** for logic extraction and contextual framing, it maps unstructured local files into a deterministic Knowledge Graph stored natively within **LanceDB**. By integrating the `Ascent` Datalog engine, Aurelius autonomously derives complex social/business relationships, executes "Compassion Protocols," and manages its own memory entropy via a daily "Metabolic Layer."

## 2. Goals / Objectives
* **Primary Objective:** Eliminate the "Lost in the Middle" hallucination risk using "Contextual Retrieval" (Document DNA prepending) and Recursive Small-to-Big retrieval.
* **Secondary Objective:** Implement a daily "Metabolic" cycle (`dream`) that autonomously curates the knowledge graph (decaying stale facts, merging duplicates, deriving transitive trust, and summarizing clusters).
* **Tertiary Objective:** Introduce "High-EQ" capabilities, allowing the system to track human-centric events (birthdays, bereavement) and override cold business logic (e.g., pausing invoice reminders via a Denial Guard).

## 3. Target Audience / Stakeholders
* **The "Power Developer":** Needs deep, highly accurate context across thousands of code files and documentation without cloud privacy risks.
* **The "Deep Researcher":** Requires a system that understands the narrative flow of complex PDFs, providing citations to exact paragraphs.
* **The "Networked Executive":** Uses the system as a Personal CRM, relying on the agent to remember human contexts (High-EQ events) to inform and gate business interactions.

## 4. User Stories / Use Cases
* **US1 (Ingestion):** As a researcher, I want the system to understand the global context of a chunk (via Document DNA prepending) so a stray paragraph about "pricing" isn't confused with a completely different project.
* **US2 (Retrieval):** As a user, I want to query specific IDs (BM25) and broad concepts (Vector) simultaneously, and have the system feed the LLM the larger surrounding narrative (Small-to-Big Retrieval) for a highly accurate answer.
* **US3 (Daily Pulse):** As an executive, I want to run `aurelius today` first thing in the morning to see a streaming dashboard of new insights, resolved contradictions, and upcoming high-EQ events.
* **US4 (HITL Training):** As a power user, I want `aurelius teach` to ask me clarifying questions about conflicting facts so I can lock in "Core Truths" that bypass the system's memory decay.

## 5. Functional Requirements

* **FR1: Contextual & Recursive Ingestion Pipeline**
  * `Kreuzberg` parses documents into Markdown.
  * Qwen generates a 100-token "Context Wrapper" (Document DNA) summarizing the file.
  * Split files into 1500-char parent blocks (`context_window_id`), subdivided into 300-char target chunks (`chunk_id`).
  * Prepend the Context Wrapper to *every* 300-char chunk.

* **FR2: Constrained Fact Extraction**
  * Pipe 300-char chunks through Qwen 3.6 (4-bit) to extract atomic facts into a strict JSON schema adhering to the Domain/Personal Ontology (see Section 6).
  * Store JSON facts, raw chunks, parent blocks, and `zembed-1` vectors in LanceDB.

* **FR3: Hybrid-Recursive Retrieval Layer**
  * Execute parallel LanceDB queries: Vector Search & Full-Text Search (BM25).
  * Merge and rank using Reciprocal Rank Fusion (RRF):
    $$Score(d) = \sum_{r \in R} \frac{1}{k + rank_r(d)}$$ *(where $k=60$)*
  * Retrieve top 300-char chunks, but pass their corresponding 1500-char *parent blocks* to the LLM for final synthesis.

* **FR4: The `dream` Command (Metabolic Cycle)**
  * **Phase A (Ascent Janitor):** * Execute pre-Ascent calendar math to compute `days_until_event`.
    * Run Datalog macros to compute Symmetry/Inversion for relationships, compute Transitive Trust, and activate Compassion Protocols (e.g., `hold_communications`).
    * Apply exponential decay to fact `Confidence`: $$Weight_{new} = Weight_{old} \times e^{-\lambda \times hours}$$. Prune facts < 0.05. Flag logical contradictions.
  * **Phase B (Qwen Synthesizer):** * Resolve flagged contradictions. Summarize clusters of weak facts into high-confidence "Neurons." Generate the LDJSON "Morning Pulse" file.

* **FR5: Active Reinforcement Interfaces**
  * **`aurelius today`:** A terminal-native streaming dashboard (built with `ratatui`) presenting the Morning Pulse (High-EQ events, health metrics, new neurons).
  * **`aurelius teach`:** An interactive Socratic prompt surfacing the top logical contradictions for user validation. Validated items are promoted to `is_core_truth = true` (Confidence locked at 1.0).

## 6. The Domain & Personal Ontology
To ensure Ascent can reason deterministically, Qwen 3.6 is restricted to extracting:

* **Entities:** `Person`, `Organization`, `Project`, `Product`, `Document`, `Concept`, `Location`, `Event`, `Invoice`, `ActionType`.
* **Predicates (Business):** `reports_to`, `manages`, `part_of`, `owns`, `requires`, `depends_on`, `authored`, `has_status`, `overdue`.
* **Predicates (Social/High-EQ):** `parent_of`, `spouse_of`, `friend_of`, `child_of`, `sibling_of`, `communicated_with`, `born_on`, `life_event`, `social_trust`, `hold_communications`.

## 7. High-EQ Business Logic (Ascent Integration)
The system bridges social and business graphs via logical derivation:
* **Symmetry & Inversion:** Automatically derives `child_of` from `parent_of`, and ensures `spouse` is bidirectional.
* **Transitive Trust:** Computes a `social_trust` lattice. Immediate family inherits high trust (0.8-0.9), which degrades multiplicatively across the network (Friend of a Friend = 0.6 * base trust).
* **Compassion Protocols (Denial Guard):** ```rust
  agent_action(ActionType::SendInvoiceReminder, client, id) :-
      invoice(id, client, "MyAgency", _),
      overdue(id, client, "MyAgency"),
      !hold_communications(client, _); // Blocks action if bereavement/life_event flagged

8. Non-Functional Requirements

    NFR1: Local Execution: Must run entirely on-device. Target hardware: Minimum 16GB RAM/VRAM utilizing Q4_K_M quantization for Qwen 3.6.

    NFR2: Determinism: The pre-Ascent calendar math and Ascent Datalog execution (Phase A of dream) must complete in under 5 seconds for a graph of up to 100,000 edges.

    NFR3: Citation Accuracy: 100% of LLM responses must trace back to a specific local file via inline citations.
